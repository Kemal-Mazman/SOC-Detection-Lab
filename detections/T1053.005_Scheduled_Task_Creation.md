Detection: Scheduled Task Creation
MITRE ATT&CK
T1053.005 – Scheduled Task/Job: Scheduled Task
Description
This detection identifies the creation of new scheduled tasks via schtasks.exe /create.
Adversaries abuse the Windows Task Scheduler to execute malicious payloads at system startup, on a recurring schedule, or in response to specific events.
This is one of the most common persistence mechanisms observed in real-world intrusions, used by threat actors including LockBit, BlackCat/ALPHV, and APT29.
Attack Simulation
To validate this detection, a scheduled task was created on the Windows 10 endpoint to simulate an attacker establishing persistence after initial access:
schtasks /create /tn "PersistenceTask" /tr "C:\Windows\System32\cmd.exe" /sc minute /mo 1 /f
This simulates an attacker setting up a recurring task that executes a payload every minute, surviving reboots and running under the creating user's context.
Data Source
Sysmon (Microsoft-Windows-Sysmon/Operational)
Event ID:
1 – Process Creation (captures schtasks.exe execution with full command line)
Splunk Detection Query
index=main sourcetype="XmlWinEventLog:Microsoft-Windows-Sysmon/Operational" "schtasks" "/create"
| rex field=_raw "CommandLine'>(?<CommandLine>[^<]+)"
| rex field=_raw "User'>(?<User>[^<]+)"
| rex field=_raw "ParentImage'>(?<ParentImage>[^<]+)"
| rex field=_raw "Image'>(?<Image>[^<]+)"
| where NOT match(CommandLine, "(?i)(\\\\Microsoft\\\\|Windows\\\\Task|SCCM|ConfigMgr)")
| table _time, User, Image, CommandLine, ParentImage
| sort - _time
Investigation Steps

Identify the user account that created the scheduled task
Review the /tr parameter — what binary or script does the task execute
Check the schedule frequency — tasks running every minute or at logon are suspicious
Determine the parent process — was schtasks spawned from cmd.exe, PowerShell, or an unusual process
Correlate with other persistence indicators (registry run keys, new services) around the same timeframe

False Positives
Possible legitimate causes include:

System administrators creating maintenance tasks
Software installers registering update tasks (e.g., Google Update, Adobe)
SCCM/ConfigMgr deploying scheduled tasks for patching
Group Policy deploying scheduled tasks across domain

Severity
Medium
Lab Validation
This detection was tested in my SOC Detection Lab by creating scheduled tasks on a Windows 10 endpoint.
Logs were ingested into Splunk via Sysmon Event ID 1 (Process Creation) using the Splunk Universal Forwarder.
