# Detection: Scheduled Task Creation

## MITRE ATT&CK
T1053.005 – Scheduled Task/Job: Scheduled Task

## Description
This detection identifies the creation of new scheduled tasks using `schtasks.exe /create`.

Adversaries commonly abuse the Windows Task Scheduler to execute malicious payloads at system startup, on a recurring schedule, or in response to specific events.

This is a widely used persistence technique observed in real-world intrusions, including ransomware groups such as LockBit and BlackCat/ALPHV, as well as APT groups like APT29.

---

## Attack Simulation
To validate this detection, a scheduled task was created on a Windows 10 endpoint:

```cmd
schtasks /create /tn "PersistenceTask" /tr "C:\Windows\System32\cmd.exe" /sc minute /mo 1 /f
```

This simulates an attacker establishing persistence by executing a payload every minute, allowing continued access even after system reboots.

---

## Data Source
**Sysmon (Microsoft-Windows-Sysmon/Operational)**

Event ID:
- **1 – Process Creation** (captures `schtasks.exe` execution with full command line)

---

## Splunk Detection Query
```spl
index=main sourcetype="XmlWinEventLog:Microsoft-Windows-Sysmon/Operational"
| rex field=_raw "<EventID>(?<EventID>\d+)</EventID>"
| rex field=_raw "CommandLine\">(?<CommandLine>[^<]+)"
| rex field=_raw "User\">(?<User>[^<]+)"
| rex field=_raw "ParentImage\">(?<ParentImage>[^<]+)"
| rex field=_raw "Image\">(?<Image>[^<]+)"
| search EventID=1 AND CommandLine="*schtasks*" AND CommandLine="*/create*"
| where NOT match(CommandLine, "(?i)(\\\\Microsoft\\\\|Windows\\\\Task|SCCM|ConfigMgr)")
| table _time, User, Image, CommandLine, ParentImage
| sort - _time
```

---

## Investigation Steps
- Identify the user account that created the scheduled task  
- Review the `/tr` parameter to determine what binary or script is executed  
- Check the task schedule (e.g., every minute or at logon is suspicious)  
- Determine the parent process (cmd.exe, PowerShell, or unusual processes)  
- Correlate with other persistence techniques (registry run keys, services)

---

## False Positives
Possible legitimate causes include:

- System administrators creating maintenance tasks  
- Software installers registering update tasks (e.g., Google Update, Adobe)  
- SCCM/ConfigMgr deployments  
- Group Policy scheduled task configurations  

---

## Severity
Medium

---

## Lab Validation
This detection was tested in a SOC detection engineering home lab by creating scheduled tasks on a Windows 10 endpoint.

Logs were collected using Sysmon (Event ID 1 – Process Creation) and forwarded to Splunk via the Splunk Universal Forwarder.
