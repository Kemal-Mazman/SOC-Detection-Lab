# Detection: Scheduled Task Creation

## MITRE ATT&CK
T1053.005 – Scheduled Task/Job: Scheduled Task

## Description
This detection identifies the creation of scheduled tasks using `schtasks.exe /create`.

Adversaries commonly abuse the Windows Task Scheduler to establish persistence by executing malicious payloads at system startup, on a recurring schedule, or in response to specific events.

This technique is widely observed in real-world intrusions and is used by threat actors such as LockBit, BlackCat/ALPHV, and APT29.

---

## Attack Simulation
To validate this detection, a scheduled task was created on a Windows 10 endpoint to simulate persistence:

```cmd
schtasks /create /tn "PersistenceTask" /tr "C:\Windows\System32\cmd.exe" /sc minute /mo 1 /f

This simulates an attacker creating a recurring task that executes a payload every minute.

Data Source

Sysmon (Microsoft-Windows-Sysmon/Operational)

Event ID:

1 – Process Creation
Splunk Detection Query
index=main sourcetype="XmlWinEventLog:Microsoft-Windows-Sysmon/Operational"
| rex field=_raw "<EventID>(?<EventID>\d+)</EventID>"
| rex field=_raw "CommandLine\">(?<CommandLine>[^<]+)"
| rex field=_raw "User\">(?<User>[^<]+)"
| rex field=_raw "ParentImage\">(?<ParentImage>[^<]+)"
| rex field=_raw "Image\">(?<Image>[^<]+)"
| search EventID=1 AND CommandLine="*schtasks*" AND CommandLine="*/create*"
| where NOT match(CommandLine, "(?i)(\\\\Microsoft\\\\|Windows\\\\Task|SCCM|ConfigMgr)")
| table _time, User, Image, CommandLine, ParentImage
| sort - _time
Investigation Steps
Identify the user account that created the scheduled task
Review the /tr parameter to determine what binary or script is executed
Check task frequency (e.g., every minute or at logon is suspicious)
Analyse the parent process (cmd.exe, PowerShell, or unusual processes)
Correlate with other persistence mechanisms (registry run keys, services)
False Positives

Possible legitimate causes include:

System administrators creating maintenance tasks
Software installers registering update tasks (e.g., Google Update, Adobe)
SCCM/ConfigMgr deploying scheduled tasks
Group Policy scheduled task deployments
Severity

Medium

Lab Validation

This detection was validated in a SOC detection engineering home lab by creating scheduled tasks on a Windows 10 endpoint.

Logs were collected using Sysmon (Event ID 1 – Process Creation) and forwarded to Splunk via the Splunk Universal Forwarder.
